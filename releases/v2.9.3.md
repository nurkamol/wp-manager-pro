# WP Manager Pro v2.9.3

**Released:** 2026-03-17

## Fixed

### 🛠️ Image Tools — GD / ImageMagick / WebP / AVIF all showing "Not available"

All four status cards on the Image Tools page showed "Not available" on production servers even though GD and ImageMagick were confirmed installed (visible in System Info → PHP).

**Root cause:** `Imagick::queryFormats('AVIF')` throws an `ImagickException` on servers where the imagick extension is loaded but lacks the AVIF codec. This exception was not caught, causing a PHP fatal error mid-response that corrupted the entire JSON payload — making every field in the response `undefined` in React, including the GD and ImageMagick flags.

**Fix:** All image library support checks now use `wp_image_editor_supports(['mime_type' => '...'])` — the same mechanism used by WordPress core and third-party image plugins (e.g. "Converter for Media"). Base GD/Imagick checks use `extension_loaded() && function_exists()/class_exists()` for reliability across all PHP SAPIs including `fpm-fcgi`.

### 🔍 Malware Scanner — false positive on own plugin files

The scanner was flagging `wp-manager-pro/includes/api/controllers/class-security-scanner-controller.php` as a "Critical" threat because the pattern-label strings (e.g. `'label' => 'eval(base64_decode(…))'`) matched the same regexes the scanner runs on other files.

**Fix:** The `wp-manager-pro` plugin directory is now unconditionally excluded from all malware scans.

## Improved

### 🔬 Malware Scanner — File Inspection & Actions

Each finding row in the Malware Scanner now has four action buttons:

| Action | Behaviour |
|--------|-----------|
| **Inspect** | Opens a modal showing the file content (±40 lines around the flagged line). The suspicious line is highlighted in red with a line number gutter. |
| **Quarantine** | Moves the file to `wp-content/wmp-quarantine/` with a `.quarantined` extension so it cannot execute. An `.htaccess` is auto-created in the quarantine root to deny all HTTP access. Requires confirmation. |
| **Delete** | Permanently removes the file from disk. Requires confirmation dialog. |
| **Ignore** | Adds the file to a persistent ignore list (`wmp_scanner_ignored` wp_options key). Ignored files are skipped in all future scans. Ignore list can be managed via the new REST endpoints. |

**New REST endpoints:**

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/scanner/file` | Read file content around a flagged line (±40 lines) |
| DELETE | `/scanner/file` | Delete a file permanently |
| POST | `/scanner/quarantine` | Move file to `wmp-quarantine/` |
| POST | `/scanner/ignore` | Add file to ignore list |
| GET | `/scanner/ignored` | List all ignored files |
| DELETE | `/scanner/ignored` | Remove a file from ignore list |

### 🏷️ Sidebar — Environment Badge Redesign

The environment badge (shown when a Dev Tools environment is set) was a flat opaque pill that looked out of place against the dark sidebar.

It is now a semi-transparent badge with a ring border and a coloured pulse dot:
- 🔴 Production — red
- 🟡 Staging — amber
- 🟢 Development — green
- 🔵 Local — blue

### 📐 Cron Manager — Spacing Normalised

The Cron Manager page wrapper, `TabsList`, and `TabsContent` spacing have been aligned with all other multi-tab pages (Security, Database, etc.):
- Wrapper: `fade-in space-y-6`
- TabsList: added `mb-2`
- TabsContent: `space-y-6 mt-2`

## Upgrade Notes

Standard plugin update. No database migrations. No breaking changes.

The new quarantine directory (`wp-content/wmp-quarantine/`) is created on first use only.
