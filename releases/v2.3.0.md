# WP Manager Pro v2.3.0 — Content Tools

**Released:** 2026-03-11

---

## Highlights

A dedicated 4-tab page for bulk content management — edit many posts at once, clone any post with one click, see all scheduled content at a glance, and directly browse/edit `wp_options` rows without touching phpMyAdmin.

---

## New Features

### 📝 Bulk Post Editor Tab
- Filter posts by **post type**, **status**, and **keyword search**
- **Checkbox** multi-select with Select All / Deselect All header toggle
- Bulk action bar appears when at least one post is selected
- Bulk-update: **Status**, **Author**, **Publish Date** across all selected posts in one API call
- Pagination (20 posts per page) with total count display
- Quick-link to edit each post directly in the WordPress editor

### 📋 Post Duplicator Tab
- Same filter/search interface as Bulk Editor
- Copy options: **Post Meta**, **Taxonomies / Categories**, **Featured Image**
- Per-row **Duplicate** button — creates a draft copy titled `"<original> (Copy)"`
- Success banner shows new draft ID with direct Edit-in-WP link
- Skips internal meta keys (`_thumbnail_id`, `_wp_old_slug`, `_edit_lock`, `_edit_last`) to avoid conflicts

### 📅 Scheduled Post Manager Tab
- Lists all posts in `future` status across all post types
- Columns: Title, Type, Scheduled Date/Time, Author, **Time Until** countdown
- Overdue scheduled posts (time already passed) highlighted in red
- Post type filter dropdown populated dynamically from posts that actually have future content
- Refresh button for manual reload
- Direct Edit link to WP editor for each post

### ⚙️ Options Table Editor Tab
- **Paginated** list of `wp_options` rows (30 per page, up to 100)
- **Search** by option name with live filter
- **Autoload filter**: show All / Yes / No
- Type detection per row: `string`, `integer`, `float`, `json`, `serialized`, `empty`
- Value preview (truncated to 80 chars; longer values show `… (N chars)`)
- **Edit dialog** — loads full raw value; `Textarea` for editing; autoload toggle; Save button
- **Delete** button (hidden for protected options: `siteurl`, `blogname`, `admin_email`, `blogdescription`, `blogpublic`, `blog_public`, `active_plugins`)
- Protected options labelled inline

## REST API — New Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/content/post-types` | List all registered public post types |
| GET | `/content/authors` | List users with `edit_posts` capability |
| GET | `/content/posts` | Paginated post list — filters: `post_type`, `status`, `search`, `per_page`, `page` |
| POST | `/content/posts/bulk-edit` | Bulk update `{ ids: int[], updates: { status?, date?, author_id?, category_id? } }` |
| POST | `/content/posts/duplicate` | Clone post `{ id, copy_meta?, copy_taxonomies?, copy_thumbnail? }` |
| GET | `/content/scheduled` | All future-scheduled posts with time-until; optional `post_type` filter |
| GET | `/content/options` | Paginated options — filters: `search`, `autoload`, `per_page`, `page` |
| GET | `/content/options/(?P<name>...)` | Full raw value of a single option |
| POST | `/content/options` | Create or update `{ option_name, option_value, autoload? }` |
| DELETE | `/content/options` | Delete `{ option_name }` — protected keys blocked with 403 |

## Technical Notes

- `class-content-controller.php` — 9 public methods + 1 private helper (`detect_option_type`)
- Duplicate detection: copies serialized meta via `maybe_unserialize` / `add_post_meta` to avoid double-serialization
- Scheduled time-until: calculated client-side from `scheduled_local` (site timezone) for accurate countdown without server round-trips
- Options editor: uses raw `$wpdb->prepare` queries (not `get_option` / `update_option`) to bypass WordPress object-cache and handle non-standard option names
- Type detection: checks serialization → empty → integer → float → JSON → string in order
