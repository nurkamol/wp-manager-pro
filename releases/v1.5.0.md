# WP Manager Pro v1.5.0

**Release date:** 2026-03-09

## Highlights

This release focuses on Image Tools — adding WebP/AVIF conversion on upload, batch conversion of existing media, and SVG sanitized uploads — plus a major Maintenance Mode upgrade with real-time countdown, scope control, and a secret bypass URL. The Dashboard gets four new Quick Actions.

---

## What's New

### 🖼️ Image Tools — WebP/AVIF Conversion on Upload

New automatic image conversion pipeline on upload:

- **Convert to WebP on Upload** — toggle in Image Settings; converts every uploaded JPEG/PNG/GIF to a `.webp` sidecar file immediately after upload
- **AVIF conversion** — also triggered on upload when enabled and server supports it
- Runs via the `wp_handle_upload` filter (fires after the file is written to disk)
- SVG, WebP, AVIF source files are skipped automatically
- Conversion uses `wp_get_image_editor()` with `GD` or `Imagick` backend

### ⚡ Image Tools — Batch Convert Existing Media

Convert the entire media library in the browser:

- **WebP row** and **AVIF row** each show `N of M images converted · K remaining`
- **Convert** button triggers paginated API calls (10 images per batch) in a loop
- **Progress bar** shows live `X / Total` counter while converting
- **Stop** button cancels mid-conversion cleanly via a `useRef` flag
- Skips images where a sidecar already exists (idempotent — safe to re-run)
- New REST routes: `POST /images/convert`, `GET /images/convert-stats`

### 🔧 Maintenance Mode — Real-Time Countdown Preview

The live preview countdown was showing static `00 DAYS / 00 HRS / 00 MIN / 00 SEC` because the implementation was using a hardcoded static array. Fixed:

- Added `previewCountdown` React state `{ d, h, m, s }`
- `useEffect` with `setInterval(tick, 1000)` calculates real remaining time from the end datetime
- Values zero-pad correctly with `String(n).padStart(2, '0')`

### 🔧 Maintenance Mode — Scope Control

Control which pages maintenance mode applies to:

- **Whole Site** (default) — all frontend requests show maintenance
- **Home Page Only** — only `is_front_page()` / `is_home()` shows maintenance
- **Specific Paths** — comma- or newline-separated path list with `*` wildcard support (uses `fnmatch()`)

### 🔑 Maintenance Mode — Secret Bypass URL

Allow developers to preview the live site during maintenance:

- **Bypass key**: 16-character alphanumeric key generated via `wp_generate_password(16, false)`
- Visiting `?wmp_preview=KEY` sets a **7-day bypass cookie** (httpOnly, same-domain, secure on HTTPS)
- Cookie validated with `hash_equals()` for timing-safe comparison
- Copy-to-clipboard button for the full bypass URL
- Regenerate button creates a new key and invalidates existing cookies

### 🏠 Dashboard — 4 New Quick Actions

Added Code Snippets, Image Tools, Email/SMTP, and Backup to the Quick Actions grid. Grid is now 3 rows × 4 columns (12 total actions).

## Bug Fixes

- **Maintenance countdown live preview showing zeros** — was rendering static `['00d', '00h', '00m', '00s']` array; `.slice(0,2)` always extracted `00`. Fixed with real-time `setInterval` countdown.
- **Snippets toggle broken** — `$wpdb->get_results()` returns string `"0"` which is truthy in JS (`!!"0" === true`). Added `(int)` cast in `get_snippets()`, `create_snippet()`, `update_snippet()`.

## Technical Notes

- **New REST routes**: `POST /images/convert`, `GET /images/convert-stats`
- **New PHP hook**: `add_filter('wp_handle_upload', [Images_Controller::class, 'convert_on_upload'])`
- **Maintenance bypass** uses `setcookie()` with `COOKIEPATH`, `COOKIE_DOMAIN`, `is_ssl()` — mirrors WordPress's own auth cookie settings
- **Scope paths** stored as newline-separated string in `wmp_maintenance_scope_paths` option; `fnmatch()` checks each path against `$_SERVER['REQUEST_URI']`
- Countdown card CSS matches PHP-generated maintenance page: `.cd-card`, `.cd-num`, `.cd-lbl` with `--accent` color variable
