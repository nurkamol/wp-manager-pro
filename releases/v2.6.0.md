# WP Manager Pro v2.6.0 — Update Manager

**Released:** 2026-03-16

---

## Highlights

A dedicated 3-tab update control centre that adds changelog preview, automatic pre-update backups, one-click rollback, and scheduled updates via WP Cron — everything the native WordPress updater lacks.

---

## New Features

### 📋 Available Updates Tab
- Lists all pending plugin, theme, and WordPress core updates
- **View Changelog** — fetches the full changelog from WordPress.org for any item before you apply the update (server-side proxy to avoid CORS)
- **Update Now** per item with animated status badge: updating → done / failed
- **Checkbox multi-select** with "Update Selected" batch action — runs sequentially to avoid file-system lock contention
- **Check for Updates** force-refresh button (triggers `wp_update_plugins()`, `wp_update_themes()`, `wp_version_check()`)
- Grouped dropdown in the Scheduled tab shows only items with pending updates (version arrow `1.2.3 → 1.4.0`)

### 🛡️ Pre-update Backup
- Every update automatically zips the plugin/theme directory to `wp-content/wmp-backups/updates/` before upgrading
- Protected by `.htaccess deny from all`
- Core WordPress updates skip file-level backup (too large); history entry still recorded

### 📜 History Tab
- Persistent log of every update run through WP Manager Pro (up to 100 entries)
- Columns: name, type badge, version arrow, date, status, rollback button
- **Rollback** button visible only when backup ZIP exists and update not already rolled back
- **Clear History** — deletes the log and all backup ZIP files

### ⏰ Scheduled Tab
- Queue any plugin or theme update for a specific future date/time
- Uses `wp_schedule_single_event` + WP Cron action `wmp_run_scheduled_update`
- Smart dropdown shows only items with pending updates, grouped by Plugins / Themes / Core
- Lists all pending jobs with next-run time; **Cancel** button removes the cron event

## REST API — New Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/updates/available` | All pending updates; `?force=1` triggers fresh WP check |
| GET | `/updates/changelog` | Fetch changelog from WordPress.org (`?type=plugin&slug=xyz`) |
| POST | `/updates/run` | Back up + upgrade a single plugin, theme, or core; logs to history |
| POST | `/updates/rollback` | Restore plugin/theme from pre-update ZIP backup |
| GET | `/updates/history` | Update history log (max 100 entries) |
| DELETE | `/updates/history/clear` | Clear history and delete all backup ZIPs |
| GET | `/updates/scheduled` | List scheduled update jobs with WP Cron next-run info |
| POST | `/updates/schedule` | Queue a plugin/theme update via `wp_schedule_single_event` |
| DELETE | `/updates/schedule/cancel` | Cancel and unschedule a queued update |

## Technical Notes

- `class-update-manager-controller.php` — uses `Plugin_Upgrader`, `Theme_Upgrader`, `Core_Upgrader` with `WP_Ajax_Upgrader_Skin`
- Backup uses `ZipArchive` + `RecursiveDirectoryIterator`
- Batch updates use sequential `for…of` + `await` on the frontend (not `Promise.all`) to avoid file-system lock contention
- History stored in `wp_options` key `wmp_update_history` (JSON array, max 100 entries with FIFO pruning)
- Scheduled jobs stored in `wp_options` key `wmp_scheduled_updates`
- Rollback: extracts backup ZIP back to `WP_PLUGIN_DIR` or `get_theme_root()`

---

## Upgrade Notes

No database migrations required. Pre-update backups are stored in `wp-content/wmp-backups/updates/` — this directory is created automatically on first use.
