# WP Manager Pro v1.6.0

**Release date:** 2026-03-09

## Highlights

This release completes the WebP image pipeline — converted sidecar files are now automatically served to supporting browsers, deleted when their originals are removed, and can optionally replace originals entirely. Inspired by the WebP Converter for Media plugin, but built-in with no extra dependencies.

---

## What's New

### 🌐 WebP Delivery — Automatic Transparent Serving

Images are now automatically served as WebP to browsers that support it:

**PHP filter (all servers):**
- New `wp_get_attachment_url` filter (`maybe_serve_webp`) checks `$_SERVER['HTTP_ACCEPT']` for `image/webp`
- If a `.webp` sidecar exists on disk, returns the `.webp` URL instead of the original
- Zero code changes required in themes — `wp_get_attachment_image()`, `get_the_post_thumbnail()`, `wp_get_attachment_url()` all benefit automatically

**Apache .htaccess (zero-PHP-overhead path):**
- Enabling "Serve WebP Automatically" writes rewrite rules into `wp-content/uploads/.htaccess` using WordPress's `insert_with_markers()`:
  ```apache
  RewriteCond %{HTTP_ACCEPT} image/webp
  RewriteCond %{REQUEST_FILENAME} \.(jpe?g|png|gif|bmp|tiff?)$
  RewriteCond %{REQUEST_FILENAME}\.webp -f
  RewriteRule ^ %{REQUEST_URI}.webp [L,T=image/webp]
  ```
- Rules are removed cleanly when the option is turned off
- **Nginx users**: the PHP filter serves as fallback; a Nginx config snippet is shown in the UI

**New toggle in Image Settings**: "Serve WebP Automatically"

### 🔄 Replace Original with WebP

New "Replace Original with WebP" option in the WebP Delivery card:

**On upload (via `wp_handle_upload` filter):**
- After converting, deletes the original JPEG/PNG
- Swaps `$upload['file']`, `$upload['url']`, and `$upload['type']` to point at the `.webp`
- WordPress receives a clean WebP attachment — no original is ever registered

**On batch convert (via `POST /images/convert?delete_original=true`):**
- For each converted image: deletes original file + all original thumbnail size files
- Updates the DB record: `update_attached_file()`, `wp_update_post(['post_mime_type' => 'image/webp'])`, `wp_update_attachment_metadata()` with freshly generated metadata from the new `.webp`
- Only supported for WebP (AVIF replace-original is not offered)

> ⚠️ **Irreversible.** A backup warning is displayed in the UI when this option is enabled.

### 🗑️ Auto-Delete Sidecar Files on Attachment Deletion

New WordPress hook: `add_action('delete_attachment', [Images_Controller::class, 'delete_sidecar_files'])`

When any attachment is deleted from the media library:
- Deletes `.webp` sidecar for the full-size image
- Deletes `.avif` sidecar for the full-size image
- Iterates `wp_get_attachment_metadata()['sizes']` and deletes all `.webp` / `.avif` variants of every registered thumbnail size (e.g. `photo-300x200.webp`, `photo-150x150.avif`)
- No orphan files are left on disk

### 🗑️ Delete All Converted Files

New trash icon button per format row in the Batch Convert card:

- Deletes all `.webp` sidecar files (or `.avif`) across the entire media library
- Confirmation dialog shows count before proceeding
- REST endpoint: `DELETE /images/convert?format=webp|avif`
- Returns `{ deleted: N }` — stats are refreshed automatically after deletion

## Bug Fixes / Improvements

- `save_as_format()` return type changed from `void` to `?string` — returns the converted file path on success, `null` on failure. Used by the replace-original logic in `convert_on_upload()`.
- `batch_convert()` now requires and includes `wp-admin/includes/image.php` before calling `wp_generate_attachment_metadata()` — prevents "function not found" errors on some configurations.

## Technical Notes

- **New REST route**: `DELETE /images/convert` → `delete_all_converted()`
- **New hooks** in `class-plugin.php`:
  - `add_action('delete_attachment', [Images_Controller::class, 'delete_sidecar_files'])`
  - `add_filter('wp_get_attachment_url', [Images_Controller::class, 'maybe_serve_webp'], 10, 2)`
- `.htaccess` insertion uses `insert_with_markers()` with marker `'WP Manager Pro WebP'` — safe to call repeatedly; idempotent
- `maybe_serve_webp` early-returns for non-raster MIME types, when serve-webp option is off, or when no Accept header is present — negligible overhead on non-image requests
- `delete_sidecar_files` uses `@unlink()` (suppressed warnings) — missing files are not an error condition
