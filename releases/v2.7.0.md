# WP Manager Pro v2.7.0 — Security Scanner

**Released:** 2026-03-16

---

## Highlights

A dedicated 4-tab Security Scanner page for proactive threat detection — malware scanning, CVE database checks, SSL certificate monitoring, and WordPress/PHP version alerts — all combined into a single animated security scorecard.

---

## New Features

### 📊 Overview Tab — Security Score
- Animated ring scorecard (0–100) with letter grade (A+ → F)
- Score computed from all completed scan results:
  - Critical malware finding: −20 pts each
  - Warning malware finding: −5 pts each
  - Critical CVE: −15 pts, High: −10 pts, Medium: −5 pts
  - SSL invalid/no HTTPS: −20 pts; expiring < 14 days: −10 pts
  - Outdated WordPress: −10 pts; EOL PHP: −15 pts; PHP EOL soon: −5 pts
- Four summary cards (Malware, Vulnerabilities, SSL, Core) — each clickable to run that individual check
- **Run All Scans** button triggers every check simultaneously

### 🔍 Malware Scanner Tab
- Scans up to **8,000 PHP/JS/HTML files** in plugins and themes directories
- **13 detection patterns:**
  - `eval(base64_decode(…))`
  - `eval(gzinflate/gzuncompress/gzdecode(…))`
  - `eval(str_rot13(…))`
  - `preg_replace` with `/e` modifier
  - `assert()` with `$_POST`/`$_GET`/`$_REQUEST`/`$_COOKIE`
  - `system()` / `exec()` / `passthru()` with user input
  - `shell_exec()` with user input
  - `create_function()` (commonly abused in exploits)
  - Long base64-encoded string (> 500 chars) in `base64_decode()`
  - Dynamic variable function call `$x($y)` with user input
  - Known webshell markers (FilesMan, WSO Shell, r57shell, c99shell, b374k)
  - `document.write(unescape(…))` JavaScript injection
  - Long hex-encoded string in `eval()`
- Files > 512 KB skipped automatically
- Scope selector: **All** / Plugins only / Themes only
- Each finding shows: file path (relative to `wp-content/`), pattern name, severity badge (critical / warning), line number, code snippet

### 🛡️ Vulnerabilities Tab
- Checks all installed plugins and themes against the **WPScan CVE API** (`wpscan.com/api/v3`)
- Configurable API key (free tier: 25 requests/day)
  - Key stored in `wp_options` as `wmp_wpscan_api_key`
  - Raw key never exposed in UI — only last 4 characters shown after saving
- Vulnerability data per item: CVE title, type, CVSS severity + score, fix version, reference links
- Only shows vulnerabilities that affect the **currently installed version** (filtered against `fixed_in`)
- Results sorted: items with vulnerabilities first
- Expandable accordion rows — items with no CVEs collapsed by default

### 🔒 SSL & Core Tab

**SSL Monitor**
- PHP `stream_socket_client` connection to site domain on port 443
- Certificate parsed via `openssl_x509_parse()`
- Displays: host, subject (CN), issuer (O), SAN entries, valid-from, valid-to, days remaining
- Colour-coded status: green (valid, > 14 days), amber (expiring < 14 days), red (invalid/expired)
- Graceful error handling for non-HTTPS sites and connection failures

**Core & PHP Version Check**
- Fetches latest WordPress version from `api.wordpress.org/core/version-check/1.7/`
- Flags outdated WordPress installations with the available version shown
- PHP EOL check with built-in end-of-life date table for PHP **5.6 through 8.4**
- Flags EOL PHP (red), near-EOL within 90 days (amber), and actively maintained (green)
- Shows MySQL/MariaDB version

## REST API — New Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/scanner/malware` | Scan PHP/JS/HTML files for malicious patterns (`?scope=plugins\|themes\|all`) |
| GET | `/scanner/vulns` | Check plugins/themes against WPScan CVE API |
| GET | `/scanner/ssl` | Verify SSL certificate for the site domain |
| GET | `/scanner/core` | Compare WP version + PHP EOL status |
| GET | `/scanner/api-key` | Return whether WPScan API key is configured (masked, last 4 chars) |
| POST | `/scanner/api-key` | Save or clear the WPScan API key |

## Technical Notes

- `class-security-scanner-controller.php` — 5 public methods + 2 private helpers
- Malware scan uses `RecursiveIteratorIterator` + `RecursiveDirectoryIterator` with `SKIP_DOTS`; capped at 8,000 files to prevent timeouts
- Each file tested against all 13 patterns; only the first matching pattern per file is reported (per-pattern, not per-file dedup)
- SSL connection: `stream_context_create` with `capture_peer_cert = true`, `verify_peer = false` (read-only cert inspection)
- WPScan API: `wp_remote_get` with `Authorization: Token token=…` header; 10-second timeout per request
- Security score computed entirely on the frontend — no extra API call required

## Upgrade Notes

No database migrations required. Navigate to **WP Manager → Security Scanner** after activating.

To use the Vulnerability Database, register for a free API key at [wpscan.com](https://wpscan.com/register) and enter it in the Vulnerabilities tab.
