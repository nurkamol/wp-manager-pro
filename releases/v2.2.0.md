# WP Manager Pro v2.2.0 — Media Manager

**Released:** 2026-03-11

---

## Highlights

A dedicated 5-tab page for keeping the WordPress media library clean and lean — find orphaned files, unused attachments, and duplicates, then compress what's left.

---

## New Features

### 🖼️ Overview Tab
- Summary cards: Total Attachments, Uploads Folder Size, Orphaned count, Unused count, Duplicate Groups
- "What Each Section Does" guide explaining each cleanup category

### ⚠️ Orphaned Tab
- Lists all attachments whose physical file is missing from disk
- Shows title, MIME type, upload date, and attachment ID
- Bulk-select checkboxes with "select all" header toggle
- Delete selected via `wp_delete_attachment( $id, true )`

### 🗂️ Unused Tab
- Lists unattached attachments (`post_parent = 0`) not used as a featured image (`_thumbnail_id`) and not referenced in any published post content (slug LIKE search)
- Thumbnail previews, file sizes, bulk-select, delete selected
- Shows total MB to be freed from current selection

### 🔁 Duplicates Tab
- Groups attachments by identical MD5 file hash
- Shows wasted space per group and grand total wasted
- First item in each group marked "Keep (original)" — subsequent copies have a Delete button
- Sorted by wasted space descending

### 🗜️ Compress Tab
- Lists all JPEG and PNG attachments with current file sizes
- Adjustable quality slider (40–100, default 82)
- Per-row "Compress" button calls `wp_get_image_editor->set_quality()->save()` in-place
- Shows before size, after size, bytes saved, and percentage saved

## REST API — New Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/media/overview` | Summary stats: total attachments, uploads size, orphaned/unused/duplicate counts |
| GET | `/media/orphaned` | List attachments with missing physical files |
| DELETE | `/media/orphaned` | Bulk delete orphaned attachments `{ ids: int[] }` |
| GET | `/media/unused` | List unattached, unreferenced attachments with thumbnails |
| DELETE | `/media/unused` | Bulk delete unused attachments `{ ids: int[] }` |
| GET | `/media/duplicates` | Group attachments by MD5 hash; returns wasted-space totals |
| DELETE | `/media/duplicate` | Delete a single duplicate attachment `{ id: int }` |
| GET | `/media/compress-candidates` | List JPEG/PNG attachments with file sizes |
| POST | `/media/compress` | Re-compress one attachment `{ id, quality? }`; returns before/after sizes |

## Technical Notes

- `class-media-controller.php` — all 9 public methods plus 4 private helpers (`count_orphaned_attachments`, `count_unused_attachments`, `count_duplicate_groups`, `dir_size`)
- Duplicate detection uses `md5_file()` on the physical file path from `get_attached_file()`
- Unused detection: excludes featured images via `_thumbnail_id` postmeta; approximates content references via basename LIKE search on `post_content`
- Compression: `wp_get_image_editor()` → `set_quality()` → `save()` to the original path; works with GD and Imagick
- Overview counts are capped (200 / 500 / 300 attachments scanned) for fast page load on large libraries
