# WP Manager Pro v2.7.1 — Bug Fixes

**Released:** 2026-03-16

---

## Highlights

Patch release fixing 6 known bugs: self-update system (GitHub Releases integration), false "Done" for premium plugins, 2FA QR code provider, custom login logout, and a plain-permalinks compatibility warning.

---

## New Features

### 🔄 Self-Update System
- `class-self-updater.php` — new file that integrates WP Manager Pro with WordPress's native plugin update mechanism
- Hooks into `pre_set_site_transient_update_plugins` to inject update data when a new GitHub release is detected
- Hooks into `plugins_api` filter to supply a plugin info / changelog modal accessible from the Plugins page
- GitHub Releases API checked with a 12-hour cache (`wmp_github_update_check` transient); failure cached for 30 minutes to avoid API rate limits
- Shows **"View Release Notes →"** link in the Plugins list update row
- Cache cleared automatically after a successful self-update via `upgrader_process_complete` action

## Bug Fixes

### 🔧 Update Manager — False "Done" for Premium / Unlicensed Plugins
- **Root cause:** `Plugin_Upgrader::upgrade()` returning `null` (no update performed due to license restriction) was treated as success by the `$success` condition
- **Fix:** Added explicit `null !== $result` check; also checks `$upgrader->skin->result` for errors captured by `WP_Ajax_Upgrader_Skin` that are not propagated as return values
- **New error codes:** `license_required` and `update_unavailable` returned from the REST API; frontend `ApiError` class exposes error codes to React components
- **New UI badges:** "License Required" (amber) and "Update Unavailable" (grey) displayed next to failed items in the Available Updates list

### 🔒 2FA QR Code Not Rendering
- **Root cause:** `setup_2fa()` used `chart.googleapis.com` which is deprecated and unreliable
- **Fix:** Switched to `https://api.qrserver.com/v1/create-qr-code/?size=200x200&ecc=M&data=…` — a free, actively maintained service; no API key required

### 🔑 Custom Login — Logout Not Working
- **Root cause:** `protect_login()` intercepted `wp-login.php?action=logout` GET requests and redirected to homepage because `'logout'` was not in the `$bypass` array; WordPress's logout handler never ran; session was never destroyed
- **Fix:** Added `'logout'` to the `$bypass` array
- **Enhancement:** New `handle_logout_redirect` method registered on the `logout_redirect` filter; after successful logout, users are redirected to the custom login URL slug (if configured) instead of `wp-login.php`

### ⚠️ Plain Permalinks Warning
- **Root cause:** With Plain permalink structure, the WordPress REST API is not accessible via `/wp-json/`, causing all plugin features to fail with network errors
- **Fix:** Added `permalinks.isPlain` flag to `wp_localize_script` data; React Layout component shows an amber warning banner across all pages when plain permalinks are detected; banner includes a direct link to `WP Admin → Settings → Permalinks`; dismissible per session

## REST API

No new endpoints. Existing `/updates/run` endpoint now returns structured error codes:
- `license_required` — plugin/theme requires a valid license to download updates
- `update_unavailable` — upgrader returned null (no update package available)
- `update_failed` — general upgrade failure

---

## Technical Notes

- `Self_Updater` caches failure responses (empty array) for 30 minutes to respect GitHub API rate limits (60 unauthenticated requests/hour)
- `Self_Updater::get_download_url()` prefers named ZIP assets (e.g. `wp-manager-pro-v2.7.1.zip`) over GitHub's auto-generated `zipball_url` (which omits `assets/build/`)
- `ApiError extends Error` added to `src/lib/api.ts`; all REST errors now carry `code` in addition to `message`

---

## Upgrade Notes

No database migrations required. This is a drop-in upgrade — install the new ZIP and activate.
