# WP Manager Pro v1.2.0

**Release date:** 2026-03-08

## Highlights

This release adds plugin & theme update management, version downgrade support, smart WP.org search buttons, and a light/dark mode toggle — all with no extra dependencies.

---

## What's New

### 🔄 Plugin & Theme Updates
- **Update button** appears (amber) on installed plugins/themes that have a pending update
- Works with standard WordPress update transients — no custom remote checks needed
- Update button also appears in WP.org search results when installed version is outdated

### 📦 Version History & Downgrade
- **History icon** button on every installed plugin/theme row
- Opens a version dialog listing all versions from WordPress.org API
- Current version highlighted with a "Current" badge
- One-click install of any older (or newer) version — uses `overwrite_package: true` for seamless replacement

### 🎨 Smart Search Buttons
- WP.org search tab now shows context-aware action buttons:
  - 🟡 **Update** — plugin/theme installed but has update available
  - ✅ **Installed** (disabled) — already at latest version
  - 🔵 **Install** — not installed yet

### 🌙 Light / Dark Mode Toggle
- **Sun/Moon button** in the sidebar footer (visible in both expanded and collapsed states)
- Persists via `localStorage` — survives page refreshes and WP admin navigation
- Full dark palette covering: main layout, page headers, plugin rows, cards, dialogs, inputs, and scrollbars

## Bug Fixes

| Issue | Fix |
|-------|-----|
| ZIP download returned 401 `rest_forbidden` | Append `&_wpnonce` query param to download URL |
| Footer "Thank you for creating with WordPress" showing | Fixed CSS selector from `.wp-manager-pro-page #wpfooter` → `#wpfooter` |
| WP.org search silently failed on local dev | Moved search from PHP (`plugins_api()`) to direct browser fetch to CORS-enabled WP.org API |
| Exported ZIP had no version in filename | Filename now uses `{slug}-{version}.zip` format |

## Technical Notes

- **New REST routes**: `POST /plugins/update`, `POST /plugins/install-version`, `POST /themes/update`, `POST /themes/install-version`
- **Theme version API**: `api.wordpress.org/themes/info/1.2/?action=theme_information&request[fields][versions]=1`
- **Plugin version API**: `api.wordpress.org/plugins/info/1.2/?action=plugin_information&request[fields][versions]=1`
- Version-specific download URLs follow the `https://downloads.wordpress.org/{type}/{slug}.{version}.zip` pattern
- Dark mode uses Tailwind `darkMode: ["class"]` with `.dark` class applied to `#wp-manager-pro-root`; theme state persisted in `localStorage` as `wmp-theme`
