# WP Manager Pro v2.9.2

**Released:** 2026-03-17

## Fixed

### Media Library picker now reliably opens the native modal
All "Select image" / "Media" buttons across the plugin (Agency Tools → Login Page logo & background, Agency Tools → Coming Soon logo & background, and Settings → Branding logo) now consistently open the native WordPress Media Library modal instead of falling back to a browser `prompt()`.

**Root cause:** `window.wp?.media` was evaluated when the React bundle first loaded — before WordPress's media scripts had fully initialised. Any check done at bundle-evaluation time would find `wp.media` undefined and silently fall back to `prompt()`.

**Fix:** A bridge function (`window.wmpOpenMedia`) is now injected via `wp_add_inline_script('wp-manager-pro', $js, 'after')` in `class-admin.php`. This runs *after* the bundle and after all `wp_enqueue_media()` dependencies are ready. React components call `window.wmpOpenMedia(title, callback)` on button click — by which time the bridge is always available.

### Settings → Branding → Custom Logo URL — Select button
The Custom Logo URL field in **Settings → Branding** now has a dedicated **Select** button (image icon) that opens the WordPress Media Library picker, consistent with the logo pickers in Agency Tools. Previously this field was a plain text input.

## Files Changed
- `includes/class-admin.php` — `wmpOpenMedia` bridge injected via `wp_add_inline_script('after')`
- `src/pages/AgencyTools.tsx` — `openMediaPicker()` updated to prefer `window.wmpOpenMedia` bridge
- `src/pages/Settings.tsx` — Custom Logo URL field now has a "Select" button using `window.wmpOpenMedia`
