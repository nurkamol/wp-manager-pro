# WP Manager Pro v2.1.0 — WP-Cron Manager

**Released:** 2026-03-10

---

## Highlights

Full control over WordPress scheduled events — browse, trigger, delete, and create custom schedules, plus a health dashboard with real-cron setup instructions.

---

## New Features

### 🕐 Event Browser
- Complete list of all scheduled cron events sorted by next-run time
- Hook name, `core` badge for WordPress core events, argument count badge
- Next-run time shown as a relative label (green = future, amber = < 5 min, red = overdue)
- Duplicate `date-fns` relative timestamp below for clarity

### ▶️ Manual Trigger
- Run any cron event on demand with one click
- Captures `ob_start` output buffer and wall-clock execution time (ms)
- Inline result banner shows success/error status, duration, and any captured output

### 🗑️ Delete Events
- Remove stuck or orphaned custom cron events
- Core WordPress events are protected — run-only (no delete button shown)
- Uses `wp_unschedule_event` for single-occurrence removal

### 📅 Custom Schedules
- Register new recurrence intervals (key, display name, interval ≥ 60 seconds)
- Persisted in `wp_options` (`wmp_custom_cron_schedules`)
- Injected into WordPress via `cron_schedules` filter on every request
- Delete custom schedules; built-in WordPress schedules cannot be removed

### 🩺 Cron Health Tab
Four status cards:
- **WP-Cron** — enabled/disabled (DISABLE_WP_CRON)
- **Overdue** — count of events past their scheduled time (with list)
- **Lock Timeout** — current WP_CRON_LOCK_TIMEOUT value
- **Alternate Cron** — ALTERNATE_WP_CRON status

Real Cron Setup guide with:
1. `wp-config.php` snippet to disable pseudo-cron
2. Server crontab command (site URL pre-filled)
3. WP-CLI command: `wp cron event run --due-now`

## REST API — New Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/cron/events` | List all scheduled events sorted by timestamp |
| POST | `/cron/run` | Trigger event immediately; returns output + duration |
| DELETE | `/cron/event` | Unschedule event (single or all occurrences) |
| GET | `/cron/schedules` | List all registered schedules (built-in + custom) |
| POST | `/cron/schedules` | Create a custom schedule |
| DELETE | `/cron/schedules` | Delete a custom schedule |
| GET | `/cron/health` | Cron health status, overdue events, real-cron hints |

## Technical Notes

- `class-cron-controller.php` — uses `_get_cron_array()` to read all events; `wp_get_schedules()` for schedule metadata
- Custom schedules hook: `add_filter('cron_schedules', [Cron_Controller::class, 'inject_custom_schedules'])`
- Manual trigger wraps `do_action_ref_array()` in `ob_start/ob_get_clean` + `try/catch` for safe output capture
- `is_core_hook()` checks against a known list of WP core hooks + `str_starts_with($hook, 'wp_')` heuristic
- `date-fns` v4 added as a runtime dependency for `formatDistanceToNow`
