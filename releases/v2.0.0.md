# WP Manager Pro v2.0.0 — Security Suite

**Release date:** 2026-03-10

## Highlights

v2.0.0 is a major milestone that transforms WP Manager Pro's Security page from a single-feature "Custom Login URL" card into a full **Security Suite** with five dedicated tabs.

---

## New Features

### Security Overview Tab
- At-a-glance status cards for all six security features (Admin URL, Login Limiter, IP Blocklist, XML-RPC, Hide WP Version, Two-Factor Auth)
- Shows WordPress version and locale with a quick link to the Integrity tab

### Login Protection Tab
- **Custom Login URL** — existing feature, now integrated into the tabbed layout
- **Login Attempt Limiter** — brute-force protection with configurable max attempts, counting window (seconds), and lockout duration (seconds); stored via WordPress transients
- **Lockout Log** — paginated history of IP lockouts with username, timestamp, and attempt count; supports per-IP unlock and bulk clear

### Hardening Tab
- **Disable XML-RPC** — applies `xmlrpc_enabled` filter to return `false`; prevents brute-force and DDoS via `xmlrpc.php`
- **Hide WordPress Version** — removes the WP version from the `<meta name="generator">` tag and strips `?ver=X.Y.Z` from script/style URLs
- **IP Blocklist** — add individual IPs or CIDR ranges (e.g. `10.0.0.0/24`) with optional notes; enforced on the `init` hook before any output; supports remove

### File Integrity Tab
- Fetches official MD5 checksums from `https://api.wordpress.org/core/checksums/1.0/`
- Compares every file in `wp-admin/` and `wp-includes/` against expected hashes
- Reports modified files (path + actual vs. expected hash + last modified date) and missing files
- `wp-content/` is excluded (user content)

### Two-Factor Authentication Tab
- TOTP-based 2FA per admin account (RFC 6238 — SHA1, 6 digits, 30-second window)
- QR code via Google Charts API for easy scanning in Google Authenticator, Authy, or any TOTP app
- Fallback manual secret entry (base32-encoded)
- 8 one-time backup codes generated on verification (shown once, stored as MD5 hashes)
- Enable / disable per-user; per-user meta keys: `wmp_2fa_secret`, `wmp_2fa_enabled`, `wmp_2fa_backup_codes`

## REST API — New Endpoints (14)

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/security/overview` | All security feature states in one call |
| POST | `/security/limiter` | Save login limiter settings |
| GET | `/security/lockouts` | List lockout log entries |
| DELETE | `/security/lockouts` | Clear all lockout log entries |
| POST | `/security/lockouts/unlock` | Unlock a specific IP |
| GET | `/security/ip-blocklist` | List blocked IPs |
| POST | `/security/ip-blocklist` | Add IP or CIDR to blocklist |
| DELETE | `/security/ip-blocklist` | Remove IP from blocklist |
| POST | `/security/hardening` | Save XML-RPC / hide-version settings |
| POST | `/security/integrity` | Run core file integrity check |
| GET | `/security/2fa` | Get 2FA status for current user |
| POST | `/security/2fa/setup` | Generate TOTP secret + QR URL |
| POST | `/security/2fa/verify` | Verify code and activate 2FA |
| DELETE | `/security/2fa` | Disable 2FA for current user |

## WordPress Hooks Added

| Hook | Type | Purpose |
|------|------|---------|
| `wp_login_failed` | action | Record failed login attempts per IP |
| `authenticate` (priority 30) | filter | Block locked-out IPs before auth completes |
| `init` | action | Check IP blocklist and block if matched |
| `xmlrpc_enabled` | filter | Disable XML-RPC (conditional) |
| `the_generator` | filter | Remove WP version meta tag (conditional) |
| `script_loader_src` | filter | Strip `?ver=X.Y.Z` from scripts (conditional) |
| `style_loader_src` | filter | Strip `?ver=X.Y.Z` from styles (conditional) |

## Technical Notes

- TOTP implementation is 100% native PHP — no Composer dependencies; uses `random_bytes`, `hash_hmac('sha1')`, and a custom base32 encoder/decoder
- IP matching supports both exact IPs and CIDR notation via bitwise mask comparison
- Lockout state stored in WordPress transients (`wmp_lockout_{md5(ip)}`, `wmp_attempts_{md5(ip)}`)
- Lockout log and IP blocklist stored in `wp_options` (capped at 500 entries for the log)
- Hardening hooks are registered conditionally at plugin init time — changes take effect on next page load

---

## Upgrade Notes

No database migrations required. All new options default to disabled/empty, so existing sites are unaffected after upgrading.
